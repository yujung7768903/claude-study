### Q1. [객관식] (출처: 20260218-websocket-in-go.md)

Go WebSocket 서버에서 클라이언트 하나가 연결되면 생성되는 고루틴은 몇 개이며, 각각의 역할은 무엇인가요?

A) 1개 — 읽기와 쓰기를 한 고루틴에서 처리
B) 2개 — `readPump`(수신), `writePump`(송신)
C) 3개 — `readPump`, `writePump`, `pingPump`
D) 4개 — 읽기, 쓰기, Ping, 에러 처리 각각 1개씩

<details>
<summary>정답 보기</summary>

**정답: B) 2개 — `readPump`(수신), `writePump`(송신)**

```go
func serveWs(hub *Hub, w http.ResponseWriter, r *http.Request) {
    go c.writePump()  // 서버 → 브라우저 전송 전담
    go c.readPump()   // 브라우저 → 서버 수신 전담
}
```

- `readPump`: 브라우저로부터 메시지를 수신하고 핸들러를 호출합니다.
- `writePump`: `send` 채널에서 메시지를 꺼내 브라우저로 전송하며, 주기적으로 Ping도 보냅니다.

Go의 고루틴은 매우 가볍기 때문에 클라이언트 수만큼 고루틴을 띄워도 부담이 없습니다.
</details>

### Q2. [주관식 - 빈칸] (출처: 20260214-k6-influxdb-metrics-and-queries.md)

k6의 4가지 메트릭 타입 중, `http_req_failed`와 `checks`가 속하는 타입은 ______이며, 이 타입의 값은 InfluxDB에 ______ 형태로 저장됩니다. InfluxDB에서 "성공률"을 구하려면 ______ 집계 함수를 사용해야 합니다.

<details>
<summary>정답 보기</summary>

**정답: `Rate` 타입 / `0` 또는 `1` (성공/실패 raw 값) / `MEAN()`**

Rate 타입은 InfluxDB에 비율이 아니라 개별 이벤트의 성공(1)/실패(0) 값이 그대로 저장됩니다.

```sql
-- 성공률을 구하는 쿼리
SELECT MEAN("value") FROM "checks"
  WHERE time > now() - 1h
  GROUP BY time(1m)
-- 예: (1+1+0+0+0)/5 = 0.4 → 40% 성공률
```

**핵심**: "Rate"는 저장 형식이 아니라 k6의 해석/집계 방식을 의미합니다. `MEAN(0/1 값) = 비율`입니다.
</details>

### Q3. [객관식] (출처: 20260205-mock-execution-flow.md)

다음 중 Mock 메서드(예: `restTemplate.exchange(...)`)가 호출되는 시점에 **실제로 실행되는 것**은 무엇인가요?

A) HTTP 네트워크 요청
B) RestTemplate 내부의 실제 exchange() 로직
C) 인자로 전달된 객체들의 생성 및 평가
D) 원격 서버와의 TLS 핸드셰이크

<details>
<summary>정답 보기</summary>

**정답: C) 인자로 전달된 객체들의 생성 및 평가**

Mock 메서드가 호출될 때:
- ✅ 서비스 코드(URL 문자열, HttpEntity, Map 생성 등)는 **모두 실행됨**
- ✅ Mock 메서드 호출 자체는 일어남
- ✅ 인자들이 실제로 평가되어 Mock에 전달되고, **호출 기록에 저장됨**
- ❌ Mock 메서드의 **내부 로직**(HTTP 요청, 네트워크 통신)은 실행되지 않음
- ✅ `when-thenReturn`에서 지정한 값만 반환됨

이렇게 저장된 인자 기록을 나중에 `verify` + `ArgumentCaptor`로 꺼내서 검증합니다.
</details>

### Q4. [주관식 - 단답형] (출처: 20260212-junit-test-method-signature.md)

JUnit 5에서 `@BeforeAll` 메서드는 기본적으로 어떤 제어자를 반드시 가져야 하나요? 또한, 이 제약을 없애려면 클래스에 어떤 어노테이션을 추가하면 되나요?

<details>
<summary>정답 보기</summary>

**정답: `static` / `@TestInstance(TestInstance.Lifecycle.PER_CLASS)`**

```java
// 기본: static 필수
@BeforeAll
static void setupOnce() {
    System.out.println("테스트 클래스 실행 전 1회");
}

// PER_CLASS 모드: static 없어도 됨
@TestInstance(TestInstance.Lifecycle.PER_CLASS)
class MyTest {
    @BeforeAll
    void setupOnce() {  // ✅ static 아니어도 됨
        System.out.println("PER_CLASS 모드에서는 가능");
    }
}
```

`@BeforeAll`/`@AfterAll`은 테스트 클래스 전체에서 1번만 실행되므로, 인스턴스 생성 전에 호출할 수 있도록 기본적으로 `static`이어야 합니다.
</details>

### Q5. [주관식 - 코드] (출처: 20260209-mockito-junit-runner-comparison.md)

JUnit 4 프로젝트에서 아래 코드의 import가 잘못되어 있습니다. 올바른 import로 수정하고, 만약 이 프로젝트를 JUnit 5로 마이그레이션한다면 어떻게 바꿔야 하는지 작성하세요.

```java
import org.mockito.runners.MockitoJUnitRunner; // ← 문제 있음

@RunWith(MockitoJUnitRunner.class)
public class MyServiceTest {
    @Mock
    private MyRepository repository;
}
```

<details>
<summary>정답 보기</summary>

**정답:**

**JUnit 4 수정 (import만 변경):**
```java
import org.mockito.junit.MockitoJUnitRunner; // ✅ org.mockito.junit 패키지

@RunWith(MockitoJUnitRunner.class)
public class MyServiceTest {
    @Mock
    private MyRepository repository;
}
```

**JUnit 5 마이그레이션:**
```java
import org.junit.jupiter.api.extension.ExtendWith;
import org.mockito.junit.jupiter.MockitoExtension;

@ExtendWith(MockitoExtension.class) // @RunWith 대신 @ExtendWith
class MyServiceTest {               // class도 public 불필요
    @Mock
    private MyRepository repository;
}
```

- `org.mockito.runners.MockitoJUnitRunner` → **Deprecated** (Mockito 2.1.0 이후)
- JUnit 4: `org.mockito.junit.MockitoJUnitRunner` 사용 (import만 변경, 나머지 코드 동일)
- JUnit 5: `@RunWith` → `@ExtendWith`, Runner → Extension으로 교체
</details>
